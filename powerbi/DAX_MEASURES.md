# Power BI DAX Measures

Every measure the dashboard needs, organized by scope. Each measure
traces to a specific visual in `DESIGN.md`. Table/column names match
the exported CSVs in `powerbi/data/`.

---

## Calculated Tables

These tables must be created in the Power BI model before measures.

### dim_date (Calculated Table)

Date dimension covering the full range of dates in the model.
Required for date slicers and the time-series area chart on Page 2.

```dax
dim_date =
VAR MinDate = MIN(fact_deductions[deduction_date])
VAR MaxDate = MAX(fact_scan_data[week_ending])
RETURN
ADDCOLUMNS(
    CALENDAR(MinDate, MaxDate),
    "year", YEAR([Date]),
    "month", MONTH([Date]),
    "year_month", FORMAT([Date], "YYYY-MM"),
    "week_ending",
        [Date] + (6 - WEEKDAY([Date], 2)),
    "month_name", FORMAT([Date], "MMM YYYY")
)
```

**Relationships:**
- `dim_date[Date]` → `fact_deductions[deduction_date]` (many-to-one)
- `dim_date[week_ending]` → `fact_scan_data[week_ending]` (many-to-one)

---

### WaterfallSteps (Disconnected Table)

Category labels for the Executive Overview waterfall chart.

```dax
WaterfallSteps =
DATATABLE(
    "Step", STRING,
    "SortOrder", INTEGER,
    {
        {"Revenue", 1},
        {"Structural Trade", 2},
        {"Operational Waste", 3}
    }
)
```

**Usage:** Waterfall chart category axis (Page 1, Visual 5). The
`SortOrder` column controls step ordering. The waterfall chart's
built-in Total bar auto-computes the net result — no "Net After
Trade" row needed.

**Sort By Column:** The semantic model sets `sortByColumn` on the
`Step` column so it sorts by `SortOrder` automatically.

### WindowWeeks (What-If Parameter Table)

Pre-period window for promo baseline calculation. Created via
New Parameter (Modeling tab) or manually.

```dax
WindowWeeks =
GENERATESERIES(1, 8, 1)
```

Column name: `WindowWeeks[WindowWeeks Value]`

**Slicer config:** Numeric slider, min 1, max 8, step 1, default 4.
Used on Page 3 (Promo Performance).

---

### TargetAllInRate (What-If Parameter Table)

Target all-in trade rate for savings calculation. Created via
New Parameter or manually.

```dax

```

Column name: `TargetAllInRate[TargetAllInRate Value]`

**Slicer config:** Percentage slider, min 0%, max 50%, step 1%,
default 18%. Used on Page 4 (Retailer Comparison).

### BaselineVolumeDynamic

- **Page(s):** Page 3
- **Visual:** Scatter chart Y-axis (replaces static IncrementalRevenue when WindowWeeks slicer is active)
- **Purpose:** Recomputes baseline using the selected WindowWeeks parameter

```dax
BaselineVolumeDynamic =
VAR SelectedWindow = [WindowWeeks Value]
VAR CurrentPromoStart = SELECTEDVALUE(dim_promo[start_week])
VAR CurrentSKU = SELECTEDVALUE(dim_promo[sku])
VAR CurrentRetailer = SELECTEDVALUE(dim_promo[retailer])
RETURN
AVERAGEX(
    FILTER(
        fact_scan_data,
        fact_scan_data[sku] = CurrentSKU
            && fact_scan_data[retailer] = CurrentRetailer
            && fact_scan_data[promo_period] = "pre"
            && fact_scan_data[week_ending] >= DATE(
                YEAR(DATEVALUE(CurrentPromoStart)),
                MONTH(DATEVALUE(CurrentPromoStart)),
                DAY(DATEVALUE(CurrentPromoStart))
            ) - (SelectedWindow * 7)
            && fact_scan_data[week_ending] < DATEVALUE(CurrentPromoStart)
    ),
    fact_scan_data[units_sold]
)
```

- **Notes:** Recalculates from raw scan data using the WindowWeeks
  parameter. Only evaluates correctly in single-promo context (Detail
  field = promo_id in scatter, or drill-through). Performance may be
  slow on 601K scan rows — consider pre-aggregating scan data by
  SKU/retailer/week if needed.

### IncrementalRevenueDynamic

- **Page(s):** Page 3
- **Visual:** Scatter chart Y-axis (when WindowWeeks ≠ 4)
- **Purpose:** Incremental revenue recalculated with dynamic baseline

```dax
IncrementalRevenueDynamic =
VAR Baseline = [BaselineVolumeDynamic]
VAR DuringAvg = AVERAGE(dim_promo[during_avg_volume])
VAR Duration = SELECTEDVALUE(dim_promo[duration_weeks], 1)
VAR ASP = SELECTEDVALUE(dim_promo[asp])
RETURN
IF(
    NOT(ISBLANK(Baseline)) && NOT(ISBLANK(DuringAvg)) && NOT(ISBLANK(ASP)),
    (DuringAvg - Baseline) * Duration * ASP,
    BLANK()
)
```

- **Notes:** Full dynamic recalculation. Use this as the scatter Y-axis
  measure to enable the what-if window slider. Falls back to BLANK when
  data is insufficient.

## Page 4: Retailer Comparison

### GrossMarginPct

- **Page(s):** Page 4
- **Visual:** Waterfall small multiples starting step; table column
- **Purpose:** Gross margin percentage per retailer

```dax
GrossMarginPct =
IF(
    HASONEVALUE(dim_retailer[retailer_name]),
    SELECTEDVALUE(dim_retailer[gross_margin], 0),
    DIVIDE(
        SUMX(dim_retailer, dim_retailer[gross_margin] * dim_retailer[revenue]),
        SUM(dim_retailer[revenue]),
        0
    )
)
```

- **Notes:** Per-retailer: reads pre-computed value from dim_retailer.
  Total row: revenue-weighted average across all retailers. The
  HASONEVALUE guard prevents the total row from returning 0.

### SavingsAtTarget

- **Page(s):** Page 4
- **Visual:** Table column (Visual 4); drives TotalSavingsAtTarget
- **Purpose:** Savings if retailer's all-in rate were reduced to target

```dax
SavingsAtTarget =
VAR CurrentAllIn = [StructuralRate] + [OpDedRate] + [PromoBBRate]
VAR Target = [TargetAllInRate Value]
VAR Rev = SELECTEDVALUE(dim_retailer[revenue], 0)
RETURN
IF(
    HASONEVALUE(dim_retailer[retailer_name]),
    IF(CurrentAllIn > Target, (CurrentAllIn - Target) * Rev, 0),
    [TotalSavingsAtTarget]
)
```

- **Notes:** Per-retailer: savings if that retailer's rate were reduced
  to target. Total row: delegates to TotalSavingsAtTarget (SUMX across
  all retailers). Updates dynamically with the TargetAllInRate slider.

### TotalSavingsAtTarget

- **Page(s):** Page 4
- **Visual:** Card (Visual 5)
- **Purpose:** Portfolio-wide savings at the selected target trade rate

```dax
TotalSavingsAtTarget =
SUMX(
    dim_retailer,
    VAR CurrentAllIn =
        CALCULATE([StructuralRate]) +
        CALCULATE([OpDedRate]) +
        CALCULATE([PromoBBRate])
    VAR Target = [TargetAllInRate Value]
    VAR Rev = dim_retailer[revenue]
    RETURN
    IF(CurrentAllIn > Target, (CurrentAllIn - Target) * Rev, 0)
)
```

- **Notes:** Iterates all retailers and sums savings. At default 18%
  target, shows how much the portfolio would save if every retailer
  above 18% were brought down to 18%.

### HighestRiskRetailer

- **Page(s):** Page 4
- **Visual:** Card (Visual 6) — risk callout
- **Purpose:** Name of the retailer with the lowest net-net margin

HighestRiskRetailer =

        FILTER(dim_retailer, dim_retailer[revenue] > 0),
        dim_retailer[net_net_margin]

RETURN

    SELECTEDVALUE(dim_retailer[retailer_name]),

## Implementation Notes

1. **Create tables first:** dim_date, WaterfallSteps, WindowWeeks,
   TargetAllInRate must exist before measures that reference them.

2. **Relationship requirements:** Measures assume active relationships
   between fact tables and dim_retailer. The BUILD_GUIDE specifies
   exact relationship configuration.

3. **Dynamic vs. static promo measures:** The export pre-computes
   promo metrics with window=4. The "Dynamic" measures recalculate
   from fact_scan_data when WindowWeeks ≠ 4. Use the dynamic variants
   as the primary scatter measures if the what-if slider is important;
   use the static variants for faster load times.

4. **Ghost promo identification:** The cleanest implementation adds an
   `is_ghost` column to fact_deductions in the export script (True for
   promo_billback deductions not matched to any planned promotion).
   This avoids complex DAX matching logic. The measures above provide
   both approaches.

5. **Format strings:** Apply in Power BI Desktop after creating measures:
   - Rates/percentages: `"0.0%"`
   - Dollar amounts: `"$#,##0"`
   - Counts: `"#,##0"`
   - ROI: `"0.00"`
